# Enchanter

シンプルなタスク管理・時間計測ツール。データはローカルのJSONファイル(`data/enchanter-data.json`)に保存されます。

## 起動方法

### 方法1: Node.jsで起動(推奨・依存パッケージなし)

Windowsは `start.cmd`、Mac/Linuxは `start.sh` をダブルクリック(または実行)するだけです。サーバーが起動し、ブラウザで http://localhost:8787 が開きます。

コマンドで起動する場合:

```
node server.js
```

### 方法2: Dockerで起動

```
docker compose up -d
```

http://localhost:8787 を開いてください。データは `./data` フォルダにマウントされ、コンテナを消しても残ります。

> **注意:** `index.html` を直接ダブルクリックしても動きません(ファイル保存にサーバーが必要なため)。必ず上記の方法で起動してください。

### 待受アドレスについて

認証機能がないため、既定では自PC(`127.0.0.1`)からのみアクセスできます。LAN内の他端末からもアクセスしたい場合は、環境変数 `HOST=0.0.0.0` を指定して起動してください(Docker利用時は `compose.yaml` のポート指定を `"8787:8787"` に変更してください)。信頼できないネットワークに公開しないよう注意してください。

## データの保存先

- `data/enchanter-data.json` に、読みやすい整形済みJSONで保存されます
- バックアップはこのファイルをコピーするだけでOKです
- 書き込みは一時ファイル経由のアトミック書き込みで、途中でクラッシュしてもデータが壊れません
- 旧バージョン(ブラウザ内localStorage保存)のデータが残っている場合、初回アクセス時に移行を提案します

## 機能

- **Todo** — タスクの追加・完了・編集・削除。クライアント/プロジェクト/重要度で絞り込み可能。各タスクの「▶ 計測」ボタンでタイマーを開始。複数タスクの並行計測に対応しており、計測中のタスクはすべてヘッダーに表示されます(2件以上のときは「すべて停止」ボタンも表示)。重要度(低/中/高)と予定日程(開始日〜終了日)を設定でき、未完了タスクは重要度が高い順、同じ重要度なら予定日が近い順に並びます。当日予定はアクセント色、期限超過は赤で表示されます。見積時間(分)を設定すると実績との対比が進捗バーで表示され、超過時は赤くなります。編集フォームからは自由記述のメモも残せます
- **タイムライン** — 日付を指定して、その日の作業記録を0〜24時の横軸で表示。その日の合計とプロジェクト別内訳、予定日程に含まれるタスクの一覧も表示。記録の手動追加・編集・削除も可能
- **ガント** — タスクの予定日程をガントチャートで表示。プロジェクトごとにグループ化され、バーはプロジェクトの色になります。表示期間は2週間/4週間/8週間から選べ、週単位で前後に移動できます。土日は網掛け、今日の列はハイライト、期限超過のタスクは赤枠で表示されます
- **集計** — 期間(開始日〜終了日)を指定して、クライアント → プロジェクト → タスクの階層で作業時間を集計。今日/今週/今月などのクイック指定あり。集計結果と作業記録の明細はCSV(Excel対応・BOM付きUTF-8)でダウンロードできます
- **管理** — クライアントとプロジェクトの登録・編集・削除。プロジェクトにはカラーを設定でき、タイムラインの色分けに使われます。プロジェクトには任意のフリーテキストIDも設定できます。「Googleカレンダー連携」カードから連携設定も行えます。「バックアップ」カードから全データのJSONエクスポート/インポート(全置き換え)ができます
- **Googleカレンダー連携**(任意) — タイマーを停止する(または作業記録を追加・編集して終了時刻が確定する)たびに、その作業記録をGoogleアカウントのメインカレンダーへ自動で反映します。詳細は下記「Googleカレンダー連携」を参照してください
- **URLにタブ状態を反映** — 表示中のタブや日付・期間は `#timeline?date=2026-07-05` のようにURLハッシュへ反映されるので、リロードしても状態が保たれ、ブックマークもできます

### キーボードショートカット

| キー | 動作 |
| --- | --- |
| `1`〜`5` | タブ切替(Todo / タイムライン / ガント / 集計 / 管理) |
| `N` | 新しいタスクを追加(Todoタブのタスク名入力へフォーカス) |
| `Esc` | 編集をキャンセル |

## API

| メソッド | パス | 内容 |
| --- | --- | --- |
| GET | `/api/data` | 全データを取得 |
| PUT | `/api/data` | 全データを保存 |
| GET | `/api/google/status` | Google連携の設定・接続状態を取得 |
| GET | `/api/google/auth-url` | Google OAuth認可URLを取得 |
| GET | `/oauth/callback` | Google OAuthのリダイレクト先(認可コード→トークン交換) |
| POST | `/api/google/disconnect` | Google連携を解除 |
| POST | `/api/calendar/sync-entry` | 1件の作業記録をGoogleカレンダーに反映(作成/更新) |

## データ構造

```json
{
  "clients": [{ "id": "...", "name": "..." }],
  "projects": [{ "id": "...", "name": "...", "customId": "...", "clientId": "...", "color": "#7c5cff" }],
  "tasks": [{ "id": "...", "title": "...", "projectId": "...", "done": false, "createdAt": 0, "completedAt": null, "plannedStart": "2026-07-05", "plannedEnd": "2026-07-07", "importance": 3, "estimateMinutes": 120, "note": "自由記述メモ" }],
  "entries": [{ "id": "...", "taskId": "...", "start": 0, "end": null }]
}
```

`entries` の `end: null` は計測中を表します。

## Googleカレンダー連携

タイマーを停止したタイミング(や、作業記録の手動追加・編集で終了時刻が確定したタイミング)で、その作業記録をGoogleアカウントのメインカレンダー(`primary`)に自動で反映する任意機能です。npmパッケージは追加せず、Node.js標準の `fetch` でGoogleのREST APIを直接呼び出します。設定しなければ通常のタスク管理・時間計測機能には一切影響しません。

### 事前準備(Google Cloud Console)

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成(または既存のものを利用)
2. 「APIとサービス」→「ライブラリ」から **Google Calendar API** を有効化
3. 「APIとサービス」→「OAuth同意画面」を設定(個人利用なら「テスト」モードで自分のGoogleアカウントをテストユーザーに追加すればOK)
4. 「認証情報」→「認証情報を作成」→「OAuthクライアントID」で、アプリケーションの種類に **「デスクトップ アプリ」** を選択して作成
5. 発行された「クライアントID」「クライアントシークレット」を `data/google-credentials.json` に次の形式で保存:

```json
{
  "client_id": "xxxxxxxx.apps.googleusercontent.com",
  "client_secret": "xxxxxxxx"
}
```

（`data/` ディレクトリはgit管理対象外なので、このファイルがリポジトリに混入する心配はありません)

### 連携方法

1. サーバーを再起動(`node server.js` または `start.cmd`/`start.sh`)
2. アプリの「管理」タブを開くと「🗓️ Googleカレンダー連携」カードが表示されます
3. 「連携する」ボタンを押し、Googleの同意画面で許可すると、アプリに戻って「連携済み」と表示されます

### 同期の挙動

- タイマー停止・作業記録の追加/編集など、作業記録の終了時刻が確定するたびに自動でGoogleカレンダーへ反映されます(手動同期ボタンはありません)
- 同じ作業記録を再度更新すると、前回作成した予定を更新します(重複作成はされません)
- 作業記録をアプリ側で削除しても、Googleカレンダー側の予定は自動削除されません(手動で削除してください)
- 「連携を解除」ボタンで、保存済みのGoogleアカウントとの接続情報を削除できます
