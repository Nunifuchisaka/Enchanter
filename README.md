# Enchanter

シンプルなタスク管理・時間計測ツール。データはローカルのJSONファイル(`data/enchanter-data.json`)に保存されます。

## 起動方法

### 方法1: Node.jsで起動(推奨・依存パッケージなし)

`start.cmd` をダブルクリックするだけです。サーバーが起動し、ブラウザで http://localhost:8787 が開きます。

コマンドで起動する場合:

```
node server.js
```

### 方法2: Dockerで起動

```
docker compose up -d
```

http://localhost:8787 を開いてください。データは `./data` フォルダにマウントされ、コンテナを消しても残ります。

> **注意:** `index.html` を直接ダブルクリックしても動きません(ファイル保存にサーバーが必要なため)。必ず上記の方法で起動してください。

## データの保存先

- `data/enchanter-data.json` に、読みやすい整形済みJSONで保存されます
- バックアップはこのファイルをコピーするだけでOKです
- 書き込みは一時ファイル経由のアトミック書き込みで、途中でクラッシュしてもデータが壊れません
- 旧バージョン(ブラウザ内localStorage保存)のデータが残っている場合、初回アクセス時に移行を提案します

## 機能

- **Todo** — タスクの追加・完了・編集・削除。プロジェクトで絞り込み可能。各タスクの「▶ 計測」ボタンでタイマーを開始。複数タスクの並行計測に対応しており、計測中のタスクはすべてヘッダーに表示されます(2件以上のときは「すべて停止」ボタンも表示)。予定日程(開始日〜終了日)を設定でき、予定日が近い順に並びます。当日予定はアクセント色、期限超過は赤で表示されます
- **タイムライン** — 日付を指定して、その日の作業記録を0〜24時の横軸で表示。その日が予定日程に含まれるタスクの一覧も表示。記録の手動追加・編集・削除も可能
- **ガント** — タスクの予定日程をガントチャートで表示。プロジェクトごとにグループ化され、バーはプロジェクトの色になります。表示期間は2週間/4週間/8週間から選べ、週単位で前後に移動できます。土日は網掛け、今日の列はハイライト、期限超過のタスクは赤枠で表示されます
- **集計** — 期間(開始日〜終了日)を指定して、クライアント → プロジェクト → タスクの階層で作業時間を集計。今日/今週/今月などのクイック指定あり
- **管理** — クライアントとプロジェクトの登録・編集・削除。プロジェクトにはカラーを設定でき、タイムラインの色分けに使われます。プロジェクトには任意のフリーテキストIDも設定できます

## 構成

| ファイル | 内容 |
| --- | --- |
| `server.js` | ローカルサーバー(静的配信 + データAPI、依存なし) |
| `index.html` | ページ骨格 |
| `style.css` | スタイル(ライト/ダークテーマ自動対応) |
| `app.js` | アプリ本体(バニラJS、依存なし) |
| `start.cmd` | Windows用起動スクリプト |
| `Dockerfile` / `compose.yaml` | Docker起動用 |

## API

| メソッド | パス | 内容 |
| --- | --- | --- |
| GET | `/api/data` | 全データを取得 |
| PUT | `/api/data` | 全データを保存 |

## データ構造

```json
{
  "clients": [{ "id": "...", "name": "..." }],
  "projects": [{ "id": "...", "name": "...", "customId": "...", "clientId": "...", "color": "#7c5cff" }],
  "tasks": [{ "id": "...", "title": "...", "projectId": "...", "done": false, "createdAt": 0, "completedAt": null, "plannedStart": "2026-07-05", "plannedEnd": "2026-07-07" }],
  "entries": [{ "id": "...", "taskId": "...", "start": 0, "end": null }]
}
```

`entries` の `end: null` は計測中を表します。
